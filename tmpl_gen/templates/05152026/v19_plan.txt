1. Why v19 exists -- reproducibility-first rebuild + equal-weight Stage 5
-------------------------------------------------------------------------

1.1 Reproducibility. v18.1 and v18.2 were built with template manifests
    sourced from earlier vintage directories (v16 -> TAA, v17.1 -> CSE)
    via implicit cross-vintage references rather than self-contained
    artifacts. The build infrastructure (_v18p1_build/, _v18_taa_build/,
    _v18_cse_build/) lived on the build host as gitignored "local
    scratch" trees. Reproducing v18.x from a clean checkout therefore
    required (a) reconstructing the cross-vintage dependency graph and
    (b) re-deriving the build watchers. v19 collapses both: the three
    manifests are co-located in 05152026/ as v19_{core,taa,cse}.txt
    (byte-identical bodies to their predecessors -- §2.1), and the
    build infrastructure ships in-repo under _v19_*_build/ so a clean
    checkout can run the full data cycle deterministically.

1.2 Equal-weight Stage 5. v18.2 (probs 0.25/0.40/0.35 over Phase A /
    Phase B / TAA) recovered RMS combined_f1 +8.4 pp (46.34 -> 54.72)
    against the v18.1-cse base but plateaued MCQ at 62.33 (gate 70.0).
    v18.2.1 attempted to lift MCQ via Phase A 0.25 -> 0.35 / Phase B
    0.45 / TAA 0.20 plus --max-samples 2400 -> 3000; it strictly
    regressed against v18.2 on 4 axes vs v18.2's 2 (MCQ 63.17, RMS
    50.37, ATE 62.40, RCM 66.80). v18.2.2 reverted to v18.2's prob mix
    with --max-samples 1500; preliminary results showed plateau on MCQ
    61.5 and a TAA Canonical regression to 2.6 (vs v16 39.5). Two
    observations from the v18.2.x series motivate the v19 Stage 5
    design:

      (a) The v18.2 probs (0.25/0.40/0.35) bias toward Phase B (catalog
          recovery) at the expense of Phase A (MCQ + KB anchor). Equal-
          weight 0.33/0.33/0.34 redistributes ~8 pp of training mass
          from Phase B to Phase A and ~2 pp to TAA. This is a small
          shift in absolute terms but removes the design-bias that
          v18.2.1 amplified in the wrong direction and v18.2.2 reverted
          conservatively.

      (b) interleave_under stops at min_source_size / max(P). With
          v18.2's max(P) = 0.40, --max-samples 2400 yielded 6000
          interleaved rows (~1500 steps). With v19's max(P) = 0.34,
          --max-samples 2400 yields 7059 rows (~1765 steps, ~17%
          longer). To preserve v18.2's wallclock and step count while
          changing only the prob mix, v19 keeps --max-samples 2400 (the
          extra ~265 steps of TAA + Phase A exposure is the intended
          consequence of equal-weighting; if it falsifies, we cap at
          --max-samples 2040 in v19.1 and rerun).

1.3 v19 is therefore RECIPE+INFRASTRUCTURE, not corpus-driven. No
    template-body changes. No row-count gate changes. The Stage 1, 2,
    3, 4 training hyperparameters carry from v18.1 verbatim. The Stage
    5 prob mix is the sole behavioural delta from v18.2.

4. Training recipe (5 stages; verbatim from v18.1 + v18.2 except §4.5)
-----------------------------------------------------------------------

4.1 Stage 1 -- Phase A (broad knowledge re-anchor; intermediate, not
    pushed). Base Qwen/Qwen2.5-14B-Instruct. Datasets:
    ift_data_2026_05_15_v19_core_a_kb_mcq_taa_soc_cm_ms_yn,
    tulu_3_sft_mixture, alpaca_en_demo. cutoff 8192, packing on, lr
    1e-5, eff_bs 16, --max-samples 240000, 1 epoch. ~8 h on 8xH100.

4.2 Stage 2 -- Phase B (RMS+ATE+VSP+RCM catalog drill). Base = Stage 1
    local checkpoint. Dataset: ift_data_2026_05_15_v19_core_b_rms_ate_
    vsp_rcm. cutoff 16384, packing off, lr 5e-6, eff_bs 8, --max-samples
    70000, 1 epoch, eval/save every 400 steps. ~5 h on 8xH100. Push:
    asg-ai/athena-cti-sft-qwen25-14b-v19-core.

4.3 Stage 3 -- TAA Classic narrow drill. Base = v19-core. Dataset:
    ift_data_2026_05_15_v19_taa. cutoff 4096, packing on, lr 5e-6,
    eff_bs 16, --max-samples 33000, 1 epoch. ~6-8 h on 8xH100. Push:
    asg-ai/athena-cti-sft-qwen25-14b-v19-taa.

4.4 Stage 4 -- CSE letter-set drill. Base = v19-taa. Dataset:
    ift_data_2026_05_15_v19_cse. cutoff 4096, packing on, lr 5e-6,
    eff_bs 16, --max-samples 33000, 1 epoch. ~4-6 h on 8xH100. Push:
    asg-ai/athena-cti-sft-qwen25-14b-v19-cse.

4.5 Stage 5 -- Recalibrate (3-shard interleaved low-LR replay; v19
    DELTA). Base = v19-cse. Datasets (interleaved, mix_strategy
    interleave_under): Phase A (prob 0.33) + Phase B (prob 0.33) +
    standalone TAA (prob 0.34). cutoff 16384, packing off, lr 1e-6
    (1/5 the Phase B catalog-install LR), eff_bs 4, --max-samples 2400
    per dataset. interleave_under stops at 2400 / 0.34 = 7059
    interleaved rows (~1765 optimizer steps; ~95-115 min on 4xH100).
    Push: asg-ai/athena-cti-sft-qwen25-14b-v19-recalibrate.

4.6 Total wall-time: ~24 h on 8xH100 for Stages 1-4; ~95-115 min
    additional on 4xH100 for Stage 5. v19-recalibrate ships as the
    headline.


5. Sign-off and falsification
-----------------------------

5.1 v19-core (after Stage 2) MUST PASS:
      CKT (MCQ) >= 70.0   (v18-1-core 67.37 baseline; lifts on v19
                           Stage 5 if it falsifies here)
      RMS       >= 55.0   (v18-1-core 57.69 baseline)
      VSP       >= 80.0   (v18-1-core 84.30 baseline)
      ATE       >= 60.0   (v18-1-core 65.60 baseline)
      RCM       >= 67.5   (v18-1-core 71.30 baseline)
      All v18-core - 2pp tolerance applied per axis.

5.2 v19-taa MUST PASS: TAA Classic >= 40.0 and v19-core axes within
    -2pp.

5.3 v19-cse MUST PASS: CSE-TI >= 34.0, CSE-Malware >= 20.0 and v19-taa
    axes within -2pp on MCQ + TAA + RCM. Stage-3 RMS / ATE / VSP
    erosion is expected (per v18.1 trajectory) and is the target of
    Stage 5.

5.4 v19-recalibrate MUST PASS: RMS >= 54.0, MCQ >= 62.0, TAA Classic
    >= 40.0, CSE-TI >= 34.0, CSE-Malware >= 20.0, ATE >= 62.0, RCM >=
    67.5, VSP >= 80.0, CyberMetric-2K >= 85.5, CyberMetric-10K >= 81.0.
    Floor set is the v18.2 §7.4 gate package carried forward.

5.5 If 5.4 passes, v19-recalibrate is the published v19 headline.
    If it misses on MCQ alone (the most likely fail mode given the
    v18.2.x history), v19.1 caps Stage 5 --max-samples to 2040 to
    match v18.2's step count and reruns. If it misses on TAA alone,
    v19.1 reverts Stage 5 probs to 0.30/0.40/0.30 (a bias-toward-Phase-B
    + TAA-cap-restored midpoint).
