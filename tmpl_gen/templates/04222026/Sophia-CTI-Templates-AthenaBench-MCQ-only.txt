AthenaBench-Aligned CTI Templates -- MCQ-only (derived from Sophia-CTI-Templates-AthenaBench-aligned.txt)

Revision 04232026-mcq-only: MCQ-focused training slate used to teach the
model the MCQ response FORMAT (not the content). The full abaligned SFT
run exhibited a collapse-to-A failure (predicted A on ~75% of rows) because
the shuffler bug left 60% of training MCQ rows ending in "Therefore, A."
This file targets ~1000 MCQ rows with a uniform A-E answer distribution.

Position-bias control:

  - Each MCQ template declares "Shuffle: mcq" so iftgen randomises option
    labels per row and rewrites the final "Therefore, X." letter. This
    relies on the tail-anchored _shuffle_mcq_options in tmpl_parser.py
    (commit 7f6dbd0+); the previous global-match shuffler silently bailed
    on ~50% of rows when CTI descriptions contained incidental "A) ..."
    bullet markers.
  - Each template declares "Count: 167" (6 * 167 = 1002 rows target).
  - After generation, validate A-E spread with a trailing count over the
    "Therefore, X." tail across the Alpaca output field. Expect a
    distribution close to 20% per letter.

Template IDs retain their original numbering (AB.MCQ.1..6) so the
generated rows are interchangeable with the parent slate.
