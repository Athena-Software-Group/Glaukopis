================================================================================
Sophia CTI Templates -- v17 (Qwen2.5-14B-Instruct, CyberSOCEval-shape narrow
                              specialist trained off the v16 TAA specialist;
                              chained narrow SFT, NOT a parallel branch)
================================================================================

This file is the self-contained template manifest for the v17 training run.
v17 is a CHAINED narrow specialist: the recipe trains a CyberSOCEval-shaped
multi-select-MCQ head on top of asg-ai/athena-cti-sft-qwen25-14b-v16 (which
itself sits on the v12 baseline). The architectural departure from the v15
parallel-branching design is intentional and bounded: v17 introduces only
JSON-letter-set output shapes that v16 does not produce, so the chained
SFT is additive at the output surface rather than competing for capacity
with the v16 TAA-attribution head. See v17_plan.txt §1 for the rationale
and risk model.

The v16 evaluation surfaced two specific gaps that v17 targets directly:
  CyberSOCEval-Malware    accuracy 10.69%  avg_score 45.15%
  CyberSOCEval-TI         accuracy 30.63%  avg_score 58.54%

Both benchmarks score with Jaccard similarity on a set of MCQ letters
emitted as a JSON object. v16 already produces the correct semantic
neighbourhood for CSE-TI questions (the +27pp lift over v12 came from the
JS.TAA.* family bump) but does not produce multi-select letter-set JSON,
which is the shape the formal CSE evaluator scores against. v17 provides
that shape across two benchmark-mirrored families:

  JS.CSE.TI.*    (CSE-TI shape):    <json_object>{"correct_answers": [...]}</json_object>
  JS.CSE.MAL.*   (CSE-Malware shape): bare {"correct_answers": [...]}

v16 -> v17 deltas (manifest only)
---------------------------------
  (1) NEW family JS.CSE.TI.{GRP,MAL,CMP,NEG}.* -- multi-select MCQs
      over MITRE ATT&CK group tradecraft with a synthetic intel-report
      narrative prefix. Output shape mirrors PurpleLlama's
      threat_intel_reasoning benchmark: <json_object>{"correct_answers":
      ["A","C"]}</json_object>. Eight templates, ~13.5K declared rows.

  (2) NEW family JS.CSE.MAL.{RPT,TAC,TGT}.* -- multi-select MCQs over
      MITRE ATT&CK malware-uses-technique with a synthetic Hybrid-
      Analysis-style JSON detonation report prefix (vx_family,
      mitre_attcks list, signatures excerpt). Output shape mirrors
      PurpleLlama's malware_analysis benchmark: bare {"correct_answers":
      ["A","B"]}. Six templates, ~9K declared rows.

  (3) NO CARRY-OVER from v16 manifest. v17 declares ZERO TAA-attribution,
      TAA-IE, TAA-NEG, CANON, MISP, RMS, MCQ, ATE, RCM, VSP, SOC, or CM
      rows. The chained SFT preserves the v16 head for those axes via
      base-model weights; the v17 shard is single-shape multi-select
      letter-set MCQs only.

  (4) RECIPE INHERITED from v15 W1 / v16 (v9-narrow): cutoff 4096,
      packing on, lr 5e-6, eff_bs 16, 1 epoch. Per-actor cap 60 carried
      over from v16. Only the base-model pointer changes from v12 ->
      v16 in SFT/autotrain/run_sft_qwen25_14b_v16_plus_v17_cse.sh.

  (5) Estimated v17 declared total: ~22,500 rows. Estimated actual yield
      after taa_actor_balance.py (per-actor cap 60) and
      dedup_against_evals.py: ~15-18K rows, ~12K CSE-TI + ~6K CSE-MAL.

  (6) ZERO CONTAMINATION. v17 emits zero rows derived from the
      CrowdStrike CyberSOCEval_data repository (Hybrid-Analysis JSONs
      and CrowdStrike PDFs are the formal eval set). The detonation-
      report and intel-report PREFIXES are entirely synthetic, generated
      from MITRE ATT&CK relationships in the local athena-cti-db Neo4j.
      The licence-allowlist gate (carried from v13/v14/v16) validates
      every emitted row carries source=athena-cti-db-internal.

The v17 corpus is licence-equivalent to v16's TAA slice: one source,
one allowlist tag (athena-cti-db-internal), zero programmatic generators.

================================================================================
Source: athena-cti-db-internal
Anchor selector {grp:intrusion-set.name} binds to MITRE ATT&CK Groups
(187 documented intrusion sets in the local Neo4j; ~4,362 group->attack
edges; ~647 group->malware edges; ~457 group->tool edges; 25 documented
campaigns). Negatives drawn from the unbound attack-pattern catalogue
(835 documented techniques) with {force ... != ...} guards for diversity.
The intel-report PREFIX synthesises a short paragraph from the group
description and a sample of its documented techniques so the model sees
the question in a long-context shape that mirrors the CSE-TI eval set.
The Answer wraps the JSON letter-set in <json_object>...</json_object>
tags; the CSE-TI postprocessor uses this wrapper as its primary parse
hook (see SFT/test/pipelines/post_processing/cybersoceval.py).

SECTION CSE-MAL-RPT -- JS.CSE.MAL.RPT.{1,2,3} multi-select malware->technique
                        (CyberSOCEval-Malware output shape; detonation-report
                         JSON context)
================================================================================
Source: athena-cti-db-internal
Anchor selector {mw:malware.name} binds to MITRE ATT&CK malware/software
(~830 documented entries in the local Neo4j; ~9,200 malware->attack-pattern
edges via the `uses` relationship). The synthetic detonation report prefix
is JSON-shaped with vx_family, sha256 (literal "hash" -- the real CSE-MAL
benchmark strips hashes via _strip_hashes before scoring), and a
mitre_attcks list seeded from the malware's documented attack-pattern set.
This trains the model to extract MITRE IDs from the JSON, match them
against the multi-choice options, and emit the bare {"correct_answers":
[...]} JSON shape that the CSE-Malware extractor parses (no <json_object>
wrapper for this family; the bare-brace path is the malware extractor's
primary parse hook -- see SFT/test/pipelines/post_processing/cybersoceval.py).
--------------------------------------------------------------------------------
