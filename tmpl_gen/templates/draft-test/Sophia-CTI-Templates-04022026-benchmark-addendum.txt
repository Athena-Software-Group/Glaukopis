Benchmark-Aligned CTI Templates (Addendum to Sophia-CTI-Templates-04022026)

This template set addresses three gaps identified against SecKnowledge / CyberPal.AI
(Levi et al., 2024, arXiv:2408.09304):

  Gap 1: Multi-hop cross-framework path templates
  Gap 2: Hard-negative / discrimination templates
  Gap 3: Task-type diversity (MCQ, Yes/No, Classification, Summarization)

ID convention:
  X.<edge>.<n>    cross-framework path  (e.g. X.VW.1 = CVE -> Weakness)
  YN.<edge>.P.n   yes/no positive
  YN.<edge>.N.n   yes/no hard-negative  (uses {force} to pick unconnected node)
  Q.<edge>.<n>    multiple-choice (A/B/C/D)
  CL.<src>.<n>    classification (closed-list)
  SU.<src>.<n>    summarization (long -> short)

Edge abbreviations:
  V = CVE          W = Weakness (CWE)      A = CAPEC
  M = Technique    T = Tactic              G = intrusion-set (threat group)
  N = Mitigation   P = EPSS                K = KEV

Note on {force}: as-written, distractor selection picks random non-match nodes.
For SecKnowledge-style adversarial distractors (semantically near the correct
answer rather than random), extend iftgen.py to pre-embed node descriptions and
filter candidate bindings by cosine similarity to the correct answer.

Section 1 - Cross-Framework Path Templates (Gap 1)

X.VW.1 Instruction: You are a CTI analyst. Map a CVE to its root-cause weakness.
Question: CVE-{cve:CVE.id} is described as: {cve:CVE.descriptions}. Which CWE is its root cause?
Answer: The root cause of CVE-{cve:CVE.id} is CWE-{cve:CVE.problemType.id} ({cve:CVE.problemType.name}). The CVE description matches the pattern defined by CWE-{cve:CVE.problemType.id}: {cve:CVE.problemType.description}.

X.VW.2 Instruction: You are a vulnerability analyst. Identify the weakness class for a CVE.
Question: Which CWE does CVE-{cve:CVE.id} fall under?
Answer: CVE-{cve:CVE.id} falls under CWE-{cve:CVE.problemType.id} ({cve:CVE.problemType.name}).

X.VW.3 Instruction: You are a CTI analyst. Provide a concrete vulnerability example of a given weakness.
Question: Give an example CVE that demonstrates CWE-{cwe:Weakness.id} ({cwe:Weakness.name}).
Answer: CVE-{cwe:Weakness.observed_as.id} is an example of CWE-{cwe:Weakness.id}. It exhibits {cwe:Weakness.description}, which is the defining characteristic of {cwe:Weakness.name}.

X.WA.1 Instruction: You are a CTI analyst linking weaknesses to attack patterns.
Question: Which CAPEC attack pattern exploits CWE-{cwe:Weakness.id} ({cwe:Weakness.name})?
Answer: CAPEC-{cwe:Weakness.related_attack_pattern.id} ({cwe:Weakness.related_attack_pattern.name}) exploits CWE-{cwe:Weakness.id}. The attack pattern leverages {cwe:Weakness.description} to achieve its goal.

X.WA.2 Instruction: You are a CTI analyst mapping attack patterns to their target weaknesses.
Question: Which CWE weakness does CAPEC-{capec:CAPEC.id} ({capec:CAPEC.name}) typically exploit?
Answer: CAPEC-{capec:CAPEC.id} typically exploits CWE-{capec:CAPEC.weakness.id} ({capec:CAPEC.weakness.name}). The attack pattern ({capec:CAPEC.description}) targets the weakness class {capec:CAPEC.weakness.name}.

X.AM.1 Instruction: You are a CTI analyst mapping attack patterns to MITRE ATT&CK techniques.
Question: Which MITRE ATT&CK technique corresponds to the adversary behaviour in CAPEC-{capec:CAPEC.id} ({capec:CAPEC.name})?
Answer: CAPEC-{capec:CAPEC.id} aligns with MITRE ATT&CK technique {capec:CAPEC.attack-pattern.mitre_id} ({capec:CAPEC.attack-pattern.name}). Both describe the same adversary behaviour: {capec:CAPEC.description}.

X.MT.1 Instruction: You are a threat intelligence analyst. Identify the tactic that a MITRE ATT&CK technique serves.
