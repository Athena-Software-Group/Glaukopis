Benchmark-Aligned CTI Templates (Addendum to Sophia-CTI-Templates-04022026)

This revision (2026-04-20) addresses two classes of errors reported against the
original addendum:

  1. Variable redefinition: per tmpl_gen/src/tmpl_gen/tmpl_parser.py, a variable
     is defined exactly once per template (Instruction + Question + Answer)
     using {var:Type.prop} or {var:existing.rel>Type.prop}. All subsequent
     references in the same template drop the ":Type" part.

  2. Schema alignment: ATT&CK mitigations are course-of-action nodes (not
     Mitigation); attack-pattern is reached from course-of-action via the
     "mitigates" relationship, so from attack-pattern we traverse the inverse
     ("mitigates<course-of-action"). CAPEC->Weakness uses "exploits" (lower-
     case). CAPEC->attack-pattern uses "map_ap". CWE->CAPEC uses
     "related_attack_pattern".

This template set still addresses the three SecKnowledge / CyberPal.AI gaps
(Levi et al., 2024, arXiv:2408.09304):

  Gap 1: Multi-hop cross-framework path templates
  Gap 2: Hard-negative / discrimination templates
  Gap 3: Task-type diversity (MCQ, Yes/No, Classification, Summarization)

ID convention:
  X.<edge>.<n>    cross-framework path  (e.g. X.VW.1 = CVE -> Weakness)
  YN.<edge>.P.n   yes/no positive
  YN.<edge>.N.n   yes/no hard-negative  (uses {force} to pick unconnected node)
  Q.<edge>.<n>    multiple-choice (A/B/C/D)
  CL.<src>.<n>    classification (closed-list)
  SU.<src>.<n>    summarization (long -> short)

Edge abbreviations:
  V = CVE          W = Weakness (CWE)      A = CAPEC
  M = Technique    T = Tactic              G = intrusion-set (threat group)
  N = course-of-action (ATT&CK mitigation) P = EPSS           K = KEV

Note on {force}: as-written, distractor selection picks random non-match nodes.
For SecKnowledge-style adversarial distractors (semantically near the correct
answer rather than random), extend iftgen.py to pre-embed node descriptions and
filter candidate bindings by cosine similarity to the correct answer.

Section 1 - Cross-Framework Path Templates (Gap 1)

X.VW.1 Instruction: You are a CTI analyst. Map a CVE to its root-cause weakness.
Question: CVE-{cve:CVE.id} is described as: {cve.descriptions}. Which CWE is its root cause?
Answer: The root cause of CVE-{cve.id} is CWE-{cve.problemType.id} ({cve.problemType.name}). The CVE description matches the pattern defined by CWE-{cve.problemType.id}: {cve.problemType.description}.

X.VW.2 Instruction: You are a vulnerability analyst. Identify the weakness class for a CVE.
Question: Which CWE does CVE-{cve:CVE.id} fall under?
Answer: CVE-{cve.id} falls under CWE-{cve.problemType.id} ({cve.problemType.name}).

X.VW.3 Instruction: You are a CTI analyst. Provide a concrete vulnerability example of a given weakness.
Question: Give an example CVE that demonstrates CWE-{cwe:Weakness.id} ({cwe.name}).
Answer: CVE-{cve:cwe.observed_as>CVE.id} is an example of CWE-{cwe.id}. It exhibits {cwe.description}, which is the defining characteristic of {cwe.name}.

X.WA.1 Instruction: You are a CTI analyst linking weaknesses to attack patterns.
Question: Which CAPEC attack pattern exploits CWE-{cwe:Weakness.id} ({cwe.name})?
Answer: CAPEC-{cap:cwe.related_attack_pattern>CAPEC.id} ({cap.name}) exploits CWE-{cwe.id}. The attack pattern leverages {cwe.description} to achieve its goal.

X.WA.2 Instruction: You are a CTI analyst mapping attack patterns to their target weaknesses.
Question: Which CWE weakness does CAPEC-{cap:CAPEC.id} ({cap.name}) typically exploit?
Answer: CAPEC-{cap.id} typically exploits CWE-{cwe:cap.exploits>Weakness.id} ({cwe.name}). The attack pattern ({cap.description}) targets the weakness class {cwe.name}.

Question: Which MITRE ATT&CK technique corresponds to the adversary behaviour in CAPEC-{cap:CAPEC.id} ({cap.name})?
Answer: CAPEC-{cap.id} aligns with MITRE ATT&CK technique {ap:cap.map_ap>attack-pattern.mitre_id} ({ap.name}). Both describe the same adversary behaviour: {cap.description}.

X.VP.1 Instruction: You are a vulnerability risk analyst. Assess exploitation likelihood using EPSS.
Question: What is the EPSS exploitation likelihood for CVE-{cve:CVE.id}, and how should this inform prioritisation?
Answer: CVE-{cve.id} has an EPSS score of {e:cve.scores<EPSS.epss} ({e.percentile} percentile), reflecting the probability of exploitation in the next 30 days. Higher scores should be prioritised for patching.

X.VK.1 Instruction: You are a SOC analyst triaging KEV-listed vulnerabilities.
Question: Is CVE-{cve:CVE.id} listed in the CISA KEV catalog, and what is the required remediation?
Answer: CVE-{cve.id} is listed in the CISA KEV catalog as {kev:cve.known_exploit>KEV.vulnerabilityName} affecting {kev.product} ({kev.vendorProject}). Required action: {kev.requiredAction}. Due by: {kev.dueDate}.

X.VWA.1 Instruction: You are a CTI analyst building an end-to-end threat picture.
Question: Trace CVE-{cve:CVE.id} from vulnerability to weakness to attack pattern.
Answer: CVE-{cve.id} ({cve.descriptions}) is an instance of CWE-{cve.problemType.id} ({cve.problemType.name}). This weakness is exploited by CAPEC-{cap:cve.impacts>CAPEC.id} ({cap.name}). Chain: CVE-{cve.id} -> CWE-{cve.problemType.id} -> CAPEC-{cap.id}.

X.TMN.1 Instruction: You are a defensive architect. Trace from a tactic to a concrete mitigation.
Question: Starting from the {tac:x-mitre-tactic.name} tactic, provide a technique that achieves it and a mitigation against that technique.
Answer: The {tac.name} tactic ({tac.description}) is achieved by technique {ap:tac.achieves<attack-pattern.mitre_id} ({ap.name}), which can be mitigated by {coa:ap.mitigates<course-of-action.mitre_id} ({coa.name}): {coa.description}.
