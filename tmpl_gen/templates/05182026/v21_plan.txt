1. Why v21 exists -- the v18.1 -> v19 -> v20 regression
-------------------------------------------------------

1.1 v18.1-core posted Total Score 58.9 (master results table).
    v19-core posted 58.5 and v20-core posted 57.9 against the same
    eval suite. Each chain Stage 1 launcher
    (run_sft_qwen25_14b_v{18p1,19,20}_core.sh) is documented as
    "byte-identical recipe to v18p1_core.sh; only dataset names,
    HF push target, and build-dir / plan references change". The
    regression is therefore in one of:
      (a) silent template-content drift between vintages;
      (b) silent row-count-gate drift between vintages;
      (c) data-build non-determinism (substrate sampling, dedup
          tiebreaks, shuffle seeding) at fixed inputs.

1.2 v21 disambiguates by forcing (a) and (b) to identity vs v18.1:
    every template body and every gate JSON in tmpl_gen/templates/
    05182026/ is a SHA-verified byte copy of its v18.1 / v16 / v17.1
    predecessor. Only paperwork (this file, README-21.md) and naming
    (file paths, dataset names, HF push targets, build dirs) differs.

1.3 If v21 reproduces v18.1 within instrument noise (§5.1), the
    v19 / v20 regression is fully attributed to dataset deltas
    documented in the v19 / v20 plans (manifest Count: bumps,
    floor lifts, etc.) and the v18.1 corpus shape is the locked
    Core target for any subsequent vintage.

1.4 If v21 misses v18.1 numbers despite byte-identical inputs, the
    regression is (c) data-build non-determinism, and the next step
    is seed-pin + sampling-stable substrate sampling in the
    watcher.sh / make_dataset.sh pipeline (out of scope for v21).

5. Sign-off and falsification
-----------------------------

5.1 v21-core (Stage 1 standalone benchmark) MUST PASS within +/- 1.5pp
    of v18.1-core on each axis (instrument-noise band):
      CKT  : v18.1-core 67.4   -> v21 target 65.9..68.9
      RMS  : v18.1-core 56.3 F1 -> v21 target 54.8..57.8
      VSP  : v18.1-core 84.3   -> v21 target 82.8..85.8
      ATE  : v18.1-core 65.6   -> v21 target 64.1..67.1
      RCM  : v18.1-core 71.3   -> v21 target 69.8..72.8
      Total: v18.1-core 58.9   -> v21 target 57.4..60.4

5.2 PASS (within band on all axes): v18.1-core recipe is bit-
    reproducible from byte-identical template + gate inputs.
    Conclude v19 / v20 regression is dataset-content driven.

5.3 FAIL on any axis with v21 < v18.1-core - 1.5pp:
    data-build non-determinism is implicated. Diagnose via
    diff-by-hash on the rebuilt ift_data_2026_05_18_v21_*.json
    files vs ift_data_2026_05_11_v18p1_*.json on the workstation;
    pin the divergent stage (substrate sampling, dedup, shuffle).

7. Outcome (recorded 2026-05-19)
--------------------------------

7.1 §5.1 gate result -- PARTIAL FAIL.
    Total Score landed inside the +/- 1.5pp band, but two per-axis
    sub-scores fell outside it. The v18.1 recipe is therefore NOT
    bit-reproducible at axis granularity from byte-identical template /
    gate inputs.
      v21-core Total : 60.8   (target 57.4..60.4)    -> PASS Total
      v21-core ATE   :        ATE delta -4.8pp        -> FAIL (>1.5pp)
      v21-core RCM   :        RCM delta -3.1pp        -> FAIL (>1.5pp)
      v21-core CKT/RMS/VSP   : within band (per-axis spot checks)
    Per §5.3 this implicates data-build non-determinism (substrate
    sampling order, dedup tiebreaks, shuffle seeding in
    tmpl_gen/data_generation/make_dataset.sh and the Neo4j read
    layer) rather than recipe drift. The v19 / v20 axis regressions
    are now best explained by the same mechanism rather than the
    Count: / floor lifts documented in their plans.

7.2 Downstream stages were run via run_sft_qwen25_14b_v21_chain.sh
    (TAA -> CSE -> Recalibrate; the latter is off-plan vs §3 and is
    included here for v19/v20 chain parity, see that script's header).
      v21-taa         : TAA Classic narrow drill, in-plan.
      v21-cse         : VSP 72.9   (Core had 82.5; -9.6pp).
                        The recurring "CSE narrow drill trades VSP
                        for CSE-MAL" pattern seen in v18.1 / v19 / v20.
      v21-recalibrate : Total 62.3, VSP recovered to 83.1.
                        Best Athena Total of the v21 chain; only
                        chain checkpoint with VSP back inside its
                        v18.1-core band.

7.3 Ship recommendation: athena-cti-sft-qwen25-14b-v21-recalibrate.
    62.3 Total beats v21-core (60.8) and v18.1-core (58.9), VSP
    recovered to 83.1, and CSE gains preserved by the low-LR
    (1e-6) 3-shard interleave at probs 0.25/0.40/0.35.

7.4 Conclusion on the §1.1 falsification matrix:
      (a) template-content drift  : RULED OUT (SHA-verified identity).
      (b) gate-content drift      : RULED OUT (SHA-verified identity).
      (c) data-build non-determinism : LIVE SUSPECT (best fit for
          the per-axis-but-not-Total drift observed at fixed
          template / gate / recipe inputs).
    Next vintage (post-v21) should pin Neo4j read order and seed
    the substrate sampler / shuffle in make_dataset.sh / watcher.sh
    before another reproducibility run is attempted.
