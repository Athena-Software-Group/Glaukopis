## 2. Base-model decision

Production v11: Qwen2.5-14B-Instruct AND Qwen2.5-32B-Instruct, in parallel.
- 14B: fastest bench loop, ships first as the v11 release candidate.
- 32B: v7 32B is the current best SFT total (62.7); v11 32B is the
  production-quality release candidate at the larger scale.

Experimental fork: Qwen3-32B (no-think), gated on harness fix landing
first. Same v11 corpus, run after the v11 14B production launch. The
thinking vs non-thinking ablation on v11-tuned weights is the novel
result for the Glaukopis paper; do NOT gate v11 release on it.

Skip v11 on Llama-3.1-8B and Foundation-Sec-8B; revisit only if v11 14B
beats v10 14B by >=3 pp on weighted total.

Mirrors v10 with renames. New build dir _v11_build/.

```bash
    tmpl_gen/templates/05032026/Sophia-CTI-Templates-v11.txt \

```

Build-side gencfg defaults for v11 (changed in tmpl_gen/data_generation/
{make_dataset,tmpl2triples}.sh: GENCONF default switched from
gencfg_default_neo4j.json to gencfg_per_primary_neo4j.json):

| gencfg key            | v10 default            | v11 default                                      | rationale                                                  |
|-----------------------|-----------------------:|-------------------------------------------------:|------------------------------------------------------------|
| per_primary_grouping  | false                  | true                                             | enables sample-with-replacement anchor diversity (below)   |
| allow_nullprops       | false                  | true                                             | tolerate null/empty/"N/A" property values on Neo4j nodes   |

Architectural fix in tmpl_gen/src/tmpl_gen/tmpl_parser.py (process_template):
when both `Sample: <var>` is declared on a template AND
per_primary_grouping=true, the generated Cypher now uses an UNWIND-based
sample-with-replacement prefix and an `ORDER BY rand() LIMIT 1` inner
CALL subquery. This fixes the v10 anchor-fixation bug where AB.MS.* and
AB.TAA.* templates collapsed to a single primary-node anchor (often zero
yield because that one anchor failed the multi-select constraint). The
old DISTINCT/LIMIT prefix is preserved as match_query_fallback.

_v11_build/watcher.sh tunables (changes from v10):

| knob                  | v10 | v11   | rationale                                                    |
|-----------------------|----:|------:|--------------------------------------------------------------|
| ACTOR_CAP             |  20 |  40   | guardrails proven; 1284 -> ~3500 attribution rows            |
| ACTOR_FLOOR           | 100 | 100   | unchanged                                                    |
| DEDUP_HIT_THRESHOLD   |   1 |   1   | unchanged                                                    |
| DEDUP_DROP_THRESHOLD  |  50 |  30   | re-inspect 100 borderline 30-50 cases before lock            |

Final dataset: SFT/data/ift_data_2026_05_03_v11.json (target ~225-235K
rows post-balance/dedup, before stratified-shuffle pass in §6.3).

### 6.3 Stratified shuffle (new utility)

LLaMA-Factory's default sampler is uniform-random over the concatenated
dataset. v10's RMS-dilution failure mode is partly attributable to
this. v11 adds a category-anchored sampler implemented in two options:

- **Option A (preferred, low-risk)**: Pre-shuffle the dataset with
  stratified interleaving. Sort rows by family, then chunk-and-interleave
  to produce a deterministic shuffle order that guarantees minimum
  representation of each eval-aligned family per N-row window (target
  N=512 = 32 effective batches). Implementation:
  tmpl_gen/scripts/stratified_shuffle.py. Output is still a flat JSON
  consumed by the standard launcher; no LLaMA-Factory changes needed.

- **Option B (fallback)**: LLaMA-Factory --mix-strategy with explicit
  weights per dataset shard. Requires splitting v11 into per-family
  shards and registering each in dataset_info.json. More plumbing,
  more configurable, easier to retune mid-experiment.

Default to A for v11. Document B as a follow-up if A under-corrects RMS.

## 7. Held-out validation slice

SFT/data/ift_data_2026_05_03_v11_val.json — 50 rows per AthenaBench
axis (RMS, ATE, VSP, RCM, MCQ, TAA-attribute, TAA-IE, TAA-NEG, MS,
plus SOC and CM proxies). Built by stratified sampling from the v11
corpus *before* the dedup pass, then verified clean of leak into train
via the same 13-gram check.

Wired into LLaMA-Factory via --eval_dataset so the run logs per-axis
eval loss every 500 steps. v10's RMS regression would have been visible
mid-training with this in place.

## 8. Evaluation plan

After training:
1. Push merged checkpoints to ${HF_USERNAME}/athena-cti-sft-qwen25-{14b,32b}-v11.
2. Register vLLM aliases athena-cti-sft-qwen25-{14b,32b}-v11-vllm in
   SFT/eval/pipelines/models.py.
3. Run full AthenaBench + CyberSOCEval + CyberMetric suites via the
   bench loop (SFT/eval/run_full_bench.sh equivalent).
4. Compare v11 vs v10 vs v9 vs v7 on:
   - per-axis accuracy (RMS, ATE, CKT, RCM, VSP, MCQ, MS,
     TAA-attribute, TAA-IE, TAA-NEG)
   - weighted total (parity with the AthenaBench summary script)
   - new TAA-canonical eval slice (alias -> canonical accuracy, scored
     separately so we can attribute any TAA gain to the right cause)
5. Run epoch-0.7 and epoch-1.0 checkpoints for the 14B; 32B at
   epoch-1.0 only unless 14B shows over-train.
6. Once Qwen3-32B harness fix lands, run v11-experimental on Qwen3-32B
   (no-think) and (thinking) with the same v11 corpus — full ablation
   table for the Glaukopis paper.

Pass criteria for v11 14B production:
- weighted total >= v9 (56.4) AND >= v10 (54.1)
- RMS >= v9 RMS (no regression on the family v8.1+ exists to protect)
- TAA-attribute >= 22% (vs v10 50.5% PLAUSIBLE; bar set on accurate)
- No axis regressed > 3 pp vs v10

Fallback if RMS specifically misses: add a v11.1 phase B with the v9
RMS-only manifest at lr 5e-6, cutoff 16384, packing off — same shape as
v9 phase B. Avoid as default; v11 single-pass with stratified shuffle
is the intended path.

## 9. Hyperparameter notes (carried from v10 unless flagged)

- LR 1e-5 cosine, 5% warmup, bf16. Drop to 1.2e-5 only if v10 loss
  curve showed late-train instability (inspect WandB before launch).
- Sequence packing on, cutoff 8192. v10 stats confirmed only 7 rows
  (0.003%) exceed this; no change.
- DeepSpeed ZeRO-3 with auto offload on <4 GPUs (14B), forced offload
  on (32B).
- save_total_limit 5, save_only_model True (matches v10).
- enable_liger_kernel True (matches v10).
- Consider warm-up LR scaling for the first 500 steps if early-batch
  RMS examples crowd the loss signal — apply only on 32B if observed.
